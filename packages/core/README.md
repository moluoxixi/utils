# Core 最佳实践

基于本项目提供的 `core` 模块（定位为业务解耦的全局通用工具函数库），以及主打**“采用极致 ESModule”**的核心设计理念，在开发和使用本模块内容时，我们推荐遵循以下最佳实践：

## 1. 坚持纯净的 ESModule
- **标准导入导出模型**：内部及外部所有的数据流转和模块引入强制使用现代化的 `import / export` 语法（拒绝历史遗留的 CommonJS `require / module.exports` 规范）。在 Node 或者浏览器环境中提供原生的加载效率和支持。
- **消除副作用 (Side-Effects)**：保证通用工具代码中除了显式的 `export` 输出外，不在全局空间中做对象挂载、不主动进行立即执行的初始化副作用。并且要在其对应 `package.json` 内声明 `"sideEffects": false`，使得打包器可以放心地静态分析。

## 2. Tree-Shaking 友好与原子化设计
- **极致的按需打包**：基于纯正的 ESM 优势，通过 Vite/Rollup 等构建体系在使用该包时，能够完美配合做到 **Tree-Shaking（摇树优化）**。开发者在使用时，必须优先使用**具名导入** (`import { utilsFn } from '@moluoxixi/core'`) 而非默认全量导入。
- **保持函数的高内聚、低耦合**：每个核心函数都必须保证单一职责，足够细化和原子化。不包含 UI 逻辑依赖或特定框架特有上下文（不依赖此项目的 Vue 或 React 存在与否）。

## 3. 严格的类型推导与测试机制（Type & Test）
- **TypeScript First 设计**：作为一个核心函数层，向外部输出必须拥有高度准确的数据结构定义和泛型（Generics）推导机制，以在编译态拦截不合理的函数传参调用现象。
- **用例覆盖 (Coverage)**：由于它是下沉到最底部的 “基础设施”，每一次函数的方法加入都应该有对应的单元测试 (Unit Tests) 守护各个边界情况，保证整个应用生态系的安全稳定运行。
